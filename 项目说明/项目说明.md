句法树生成器 (Syntax Tree Generator) 项目规格说明书

1. 项目概述

本项目旨在开发一个基于 Web 的句法树（Syntax Tree）生成工具，功能类似于 rsyntaxtree。用户可以通过在网页端输入轻量级的括号表达式（如 [S [NP A dog]]），一键生成高质量的句法树矢量图 (SVG 格式)。
系统将采用前后端分离的设计思想，后端基于 FastAPI 构建，并通过 Docker 容器化技术整合 LaTeX 编译与图像转换工具，实现高度封装与一键部署。

2. 核心功能需求

单页面工作台：提供一个简洁的首页 UI。

代码输入区：允许用户输入类似 [S [NP A dog]] 的文本代码。

触发机制：提供一个“生成 (Generate)”按钮。

结果展示区：在按钮下方提供一个画板区域，用于直接渲染并展示生成的 SVG 格式句法树图像。

3. 技术栈选型

后端框架：FastAPI (Python)

前端技术：HTML + CSS + JavaScript (原生或轻量级框架，由 FastAPI 的 Jinja2 模板直接渲染或作为静态文件提供)

排版引擎：LaTeX (推荐使用 tikz-qtree 或 forest 宏包来解析括号表达式)

格式转换：pdf2svg 或 poppler-utils (用于将 PDF 高保真转换为 SVG)

容器化：Docker (用于封装 Python 环境、LaTeX 环境及系统级转换工具)

4. 系统工作流程 (Workflow)

当用户在前端点击“生成”按钮后，系统将执行以下闭环流程：

请求接收：FastAPI 后端接收到前端发来的 POST 请求，提取其中的括号表达式字符串。

模板渲染：后端读取预先准备好的 LaTeX 模板文件 (template.tex)，将括号表达式动态注入到模板的指定位置。

编译 PDF：在 Docker 容器内部调用 LaTeX 编译器（如 xelatex 或 pdflatex），将生成的 .tex 文件编译为 .pdf 文件。

转换 SVG：调用系统工具（如 pdf2svg），将上一步生成的 .pdf 转换为无损的 .svg 图像。

响应返回：FastAPI 读取生成的 SVG 文件内容，将其作为 HTTP 响应（或 JSON 数据的一部分）返回给前端。

前端渲染：前端接收到 SVG 数据后，将其直接插入到 DOM 中，展示给用户。

清理工作：后台删除本次请求产生的临时文件（.tex, .aux, .log, .pdf, .svg），释放存储空间。

5. 容器化设计 (Docker)

Docker 镜像是本项目的核心，需要确保包含所有必要的运行环境：

基础镜像：推荐使用 python:3.10-slim 或类似体积较小的镜像。

系统依赖安装：

texlive-latex-base, texlive-fonts-recommended, texlive-latex-extra (提供 LaTeX 编译能力和绘图宏包)

pdf2svg (提供 PDF 到 SVG 的转换能力)

Python 依赖：fastapi, uvicorn, jinja2 等。

启动命令：针对开发环境，使用支持热重载的命令：

uvicorn main:app --host 0.0.0.0 --port 8000 --reload


注：为了让 --reload 在 Docker 中生效，需要将宿主机的代码目录通过 Volumes 挂载到容器内部。

6. API 接口设计 (暂定)

6.1 获取首页

路径: /

方法: GET

功能: 返回前端 HTML 页面。

6.2 生成句法树

路径: /api/generate

方法: POST

请求体 (JSON):

{
  "code": "[S [NP A dog]]"
}


响应体:

成功 (200 OK): 返回 SVG 的原始文本数据，或者包含 SVG 文本的 JSON：

{
  "status": "success",
  "svg_content": "<svg>...</svg>"
}


失败 (400/500): 返回 LaTeX 编译错误或系统错误信息，以便前端提示用户检查语法。
