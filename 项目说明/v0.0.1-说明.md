# Syntaxtree v0.0.1 说明

## 版本信息

- **版本号**: 0.0.1
- **发布日期**: 2026-03
- **项目定位**: 基于 Web 的高精度句法树生成工具（类似 rsyntaxtree，但底层更强大）

## 项目概述

Syntaxtree 是一款轻量级、高性能的句法树生成器。用户在网页端输入标准的 PTB 风格括号表达式（如 `[S [NP [NN [dog]]] [VP [VB [runs]]]]`），即可一键生成高质量、文本可复制的句法树矢量图（SVG 格式）。本项目完美解决了传统 LaTeX 转 Web 图像时的文本矢量化（Path 坍缩）问题。

## 核心功能

| 功能 | 说明 |
|------|------|
| 括号表达式输入 | 严格支持 LaTeX forest 宏包风格的括号表示法，如 `[S [NP [DT [A]] [NN [dog]]]]`。 |
| 标点节点防弹衣 | 自动拦截并转义 `[,]` 节点，底层注入 `{,}` 保护层，完美解决逗号被 TikZ 引擎误认为参数分隔符的终极天坑。 |
| 原生 SVG 渲染 | 摒弃传统 PDF 转换路线，直接生成包含 `<text>` 标签和内联 WOFF2 字体的纯正 SVG，文本可完美选中与复制。 |
| 沉浸式灯箱预览 | 点击预览区即可打开灯箱，支持鼠标左键拖拽平移、滚轮精准缩放（以鼠标指针为中心）。 |
| 独立视图与复制 | 提供「在新标签页打开 SVG」功能，便于大图独立查看；支持一键复制完整 SVG 源码到剪贴板。 |
| 性能监控 | 界面右下角实时展示本次生成的编译耗时。 |

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | FastAPI + Python 3.12（提供极速的高并发 API 响应） |
| 前端 | 原生 HTML + CSS + Vanilla JavaScript（零负担，轻量化） |
| 排版引擎 | XeLaTeX + forest 宏包（学术界公认的最高质量树状图引擎） |
| 矢量转换 | dvisvgm（直接读取 XDV 生成 SVG，完美保留文本与字体结构） |
| 容器化 | Docker (python:3.12-slim + 最小化 TeX Live 环境) |

## 工作流程

```
用户前端输入括号表达式
       ↓
后端进行标点转义（处理逗号陷阱）+ 反斜杠安全过滤
       ↓
注入预设的 LaTeX 模板 (template.tex，挂载 dvisvgm 驱动)
       ↓
执行 xelatex -no-pdf → 生成轻量级 XDV 中间文件
       ↓
执行 dvisvgm --font-format=woff2 --exact --bbox=min → 渲染最终 SVG
       ↓
后端读取 SVG 内容并返回给前端渲染
       ↓
异步清理工作目录临时文件 (确保容器健康)
```

## 安全措施

- **反斜杠阻断**：严格将输入中的 `\` 替换为 `\textbackslash `，从物理层面彻底阻断 `\input`、`\include` 等恶意读取宿主机系统文件的操作。
- **逗号安全转义**：正则匹配 `[,]` 并替换为 `[{,}]`，仅作用于 forest 语法节点，无任何命令注入风险。
- **沙箱隔离**：LaTeX 编译过程被完全限制在 Docker 容器内部运行，且系统的 `\write18`（执行外部系统命令）已被严格禁用。

## 快速开始

### 1. 构建应用镜像

```bash
docker build -t syntaxtree-app .
```

### 2. 创建并运行容器（开发模式：将本地 app 目录挂载进容器以支持热更新）

```bash
docker run -it --name syntaxtree -p 8000:8000 -v ${PWD}/app:/app/app syntaxtree-app
```

### 3. 打开浏览器访问

```
http://localhost:8000
```

## API 接口参考

### POST /api/generate

**请求体 (JSON)**

```json
{
  "code": "[S [NP [NN [dog]]] [VP [VB [runs]]]]"
}
```

**成功响应 (HTTP 200)**

```json
{
  "status": "success",
  "svg_content": "<svg xmlns=\"http://www.w3.org/2000/svg\" ...>...</svg>"
}
```

**失败响应 (HTTP 400/500)**

```json
{
  "detail": "LaTeX 编译失败: 具体的排版错误日志输出..."
}
```

## 已知限制

- 暂不支持跨页的多页 SVG 生成（通常情况下 forest 的单页画布已能满足绝大多数长难句的渲染需求）。
- SVG 的初始渲染依赖容器内安装的 LaTeX 基础字体。
- 编译错误信息目前仅在 SVG 展示区进行抛出，左侧控制面板暂不提供重复的交互式错误提示。
